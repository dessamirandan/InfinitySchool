Apostila Flexbox: Dominando o Layout Flexível

Seja bem-vindo à sua apostila completa sobre Flexbox! Este guia foi cuidadosamente elaborado para te ajudar a entender e dominar o modelo de layout CSS que revolucionou a forma como construímos interfaces responsivas e organizadas na web.

O Flexbox, ou Flexible Box Module, é uma especificação CSS3 que oferece uma maneira mais eficiente de dispor, alinhar e distribuir espaço entre itens em um contêiner, mesmo quando seu tamanho é desconhecido ou dinâmico. Ele é ideal para layouts de componentes de aplicações, pequenas telas e quando você precisa de controle preciso sobre a organização de elementos em uma única dimensão (linha ou coluna).

---

Introdução ao Flexbox

Antes do Flexbox, centralizar elementos ou criar colunas com altura igual era um desafio. Com o Flexbox, esses problemas são resolvidos de forma elegante e intuitiva. Ele funciona com base em dois conceitos principais:

  • Contêiner Flex (Flex Container): O elemento pai que se torna um contêiner flex ao ter a propriedade display: flex; ou display: inline-flex; aplicada. É ele quem define o contexto flex para seus filhos diretos.
  • Itens Flex (Flex Items): Os elementos filhos diretos do contêiner flex. São eles que serão dispostos, alinhados e terão seu espaço distribuído conforme as propriedades definidas no contêiner ou neles próprios.

Pense no Flexbox como um conjunto de ferramentas para organizar itens em uma linha ou coluna de forma flexível. Você define como esses itens devem se comportar dentro de um espaço limitado ou ilimitado.

---

1. Propriedade `display`

A propriedade **`display`** é o ponto de partida para qualquer layout Flexbox. Ela é a chave que transforma um elemento HTML comum em um **Contêiner Flex**, permitindo que ele gerencie a disposição de seus filhos diretos (os itens flex) de uma maneira flexível e responsiva.

Para ativar o Flexbox em um elemento, você pode usar um dos dois valores para a propriedade `display`:

1.1. `display: flex;`

Quando você aplica `display: flex;` a um elemento, ele se comporta como um **elemento de bloco** em relação aos seus irmãos no fluxo normal do documento. Isso significa que ele ocupará 100% da largura disponível do seu pai (a menos que uma largura explícita seja definida) e iniciará em uma nova linha.

Internamente, porém, ele se torna um contêiner flex, e seus filhos diretos se tornam itens flex, sendo dispostos lado a lado por padrão (flex-direction: row;).

.meu-container-bloco {
    display: flex;
    /* Agora, seus filhos diretos serão itens flexíveis */
}

Exemplo: `display: flex;`

1.2. `display: inline-flex;`

Ao usar `display: inline-flex;`, o elemento se comporta como um **elemento em linha** em relação aos seus irmãos no fluxo normal do documento. Isso significa que ele tentará ocupar apenas o espaço necessário para seu conteúdo e permitirá que outros elementos em linha (ou `inline-block`) fiquem ao seu lado na mesma linha.

Assim como `display: flex;`, ele internamente transforma seus filhos diretos em itens flex, dispostos em linha por padrão.

    display: inline-flex;
    /* Seus filhos diretos também serão itens flexíveis */
}

1.3. Por que isso é importante?

A escolha entre `display: flex;` e `display: inline-flex;` depende do contexto do seu layout:

  • Use **`display: flex;`** quando você quer que o contêiner flex ocupe sua própria "linha" no layout principal e controle a disposição de seus filhos internamente. É ideal para seções grandes, barras de navegação principais ou componentes de layout de bloco.
  • Use **`display: inline-flex;`** quando você precisa que o contêiner flex se comporte como parte do fluxo de texto ou de outros elementos em linha, mas ainda assim deseja controlar a disposição de seus filhos com Flexbox. É útil para botões com ícones, grupos de tags, ou pequenos componentes dentro de uma linha de texto.

Entender essa distinção fundamental é o primeiro passo para dominar o Flexbox, pois ele define como o seu contêiner flex interage com o restante do documento.

2. `flex-direction`

A propriedade **`flex-direction`** define o **eixo principal** ao longo do qual os itens flex são dispostos dentro do contêiner flex. Em outras palavras, ela determina se os itens serão organizados em uma linha (horizontalmente) ou em uma coluna (verticalmente).

Os valores possíveis são:

  • row (padrão): Os itens são dispostos em uma linha, na direção do texto (da esquerda para a direita em idiomas LTR como o português).
  • row-reverse: Os itens são dispostos em uma linha, mas na direção oposta ao texto (da direita para a esquerda em LTR).
  • column: Os itens são dispostos em uma coluna, de cima para baixo.
  • column-reverse: Os itens são dispostos em uma coluna, de baixo para cima.
.container {
    display: flex;
    flex-direction: row; /* Valores comuns: row | row-reverse | column | column-reverse */
}

3. `flex-wrap`

A propriedade **`flex-wrap`** controla como os itens flex se comportam quando não há espaço suficiente para todos eles em uma única linha (ou coluna, dependendo do `flex-direction`). Ela define se os itens devem quebrar para a próxima linha/coluna ou se devem permanecer em uma única, possivelmente transbordando o contêiner.

Os valores possíveis são:

  • nowrap (padrão): Os itens permanecem em uma única linha. Se o espaço for insuficiente, os itens encolherão (se `flex-shrink` permitir) ou transbordarão o contêiner.
  • wrap: Os itens quebram para a próxima linha (ou coluna) quando não há espaço suficiente, começando do início da nova linha/coluna.
  • wrap-reverse: Os itens quebram para a linha (ou coluna) anterior, ou seja, na direção oposta ao `wrap` normal.
.container {
    display: flex;
    flex-wrap: wrap; /* Valores comuns: nowrap | wrap | wrap-reverse */
}

Exemplo: `flex-wrap: nowrap;` (Padrão)

4. `flex-flow`

A propriedade **`flex-flow`** é uma propriedade de atalho (shorthand) que combina as propriedades `flex-direction` e `flex-wrap` em uma única declaração. Ela permite definir tanto a direção dos itens quanto o comportamento de quebra de linha de forma concisa.

A sintaxe é simplesmente:

.container {
    flex-flow: <flex-direction> <flex-wrap>;
}

Por exemplo, `flex-flow: row wrap;` é o mesmo que:

.container {
    flex-direction: row;
    flex-wrap: wrap;
}

Se um dos valores for omitido, o valor padrão da propriedade correspondente será utilizado (`row` para `flex-direction` e `nowrap` para `flex-wrap`).

Exemplo: `flex-flow: column wrap;`

5. `justify-content`

A propriedade **`justify-content`** alinha os itens flex **ao longo do eixo principal** do contêiner. A direção do eixo principal é definida por `flex-direction`.

Esta propriedade é fundamental para distribuir o espaço extra entre os itens ou ao redor deles.

Os valores possíveis são:

  • flex-start (padrão): Os itens são agrupados no início do eixo principal.
  • flex-end: Os itens são agrupados no final do eixo principal.
  • center: Os itens são agrupados no centro do eixo principal.
  • space-between: O primeiro item fica no início, o último no final, e o espaço restante é distribuído igualmente **entre** os itens.
  • space-around: Os itens recebem espaço igualmente em ambos os lados. Isso resulta em um espaço menor nas extremidades do contêiner e um espaço maior entre os itens.
  • space-evenly: O espaço entre os itens e entre os itens e as bordas do contêiner é igual.
.container {
    display: flex;
    justify-content: center; /* Valores comuns: flex-start | flex-end | center | space-between | space-around | space-evenly */
}

6. `align-items`

A propriedade **`align-items`** alinha os itens flex **ao longo do eixo transversal** (perpendicular ao eixo principal) do contêiner flex. Ela controla como os itens são alinhados em sua linha flexível.

Os valores possíveis são:

  • stretch (padrão): Os itens esticam para preencher o contêiner ao longo do eixo transversal, respeitando min-height/max-height (ou `min-width`/`max-width` se `flex-direction` for `column`).
  • flex-start: Os itens são agrupados no início do eixo transversal.
  • flex-end: Os itens são agrupados no final do eixo transversal.
  • center: Os itens são agrupados no centro do eixo transversal.
  • baseline: Os itens são alinhados pela linha de base de seu conteúdo (o texto ou outros elementos inline internos).
.container {
    display: flex;
    align-items: stretch; /* Valores comuns: stretch | flex-start | flex-end | center | baseline */
    height: 150px; /* Importante para visualizar o alinhamento vertical */
}

7. `align-content`

A propriedade **`align-content`** alinha as **linhas flexíveis** ao longo do eixo transversal do contêiner flex. Ela é usada quando o contêiner tem várias linhas de itens (ou seja, quando `flex-wrap` está definido como `wrap` ou `wrap-reverse`) e há espaço extra no eixo transversal.

**Importante:** Esta propriedade não tem efeito quando há apenas uma única linha de itens flex.

Os valores possíveis são:

  • stretch (padrão): As linhas esticam para preencher o espaço disponível.
  • flex-start: As linhas são agrupadas no início do eixo transversal.
  • flex-end: As linhas são agrupadas no final do eixo transversal.
  • center: As linhas são agrupadas no centro do eixo transversal.
  • space-between: A primeira linha fica no início, a última no final, e o espaço restante é distribuído igualmente **entre** as linhas.
  • space-around: O espaço é distribuído igualmente ao redor de cada linha, com as extremidades do contêiner recebendo metade do espaço.
  • space-evenly: O espaço entre as linhas e entre as linhas e as bordas do contêiner é igual.

8. Propriedades dos Itens Flex

Enquanto as propriedades anteriores são aplicadas ao **Contêiner Flex**, as propriedades que veremos agora são aplicadas diretamente aos **Itens Flex** (os filhos diretos do contêiner). Elas permitem um controle individual sobre o comportamento de cada item.

8.1. `order`

A propriedade **`order`** define a ordem em que um item flex é exibido dentro do contêiner. Por padrão, todos os itens têm um `order` de `0` e são exibidos na ordem em que aparecem no HTML.

Itens com valores de `order` menores são exibidos antes dos itens com valores maiores. Se dois itens tiverem o mesmo valor de `order`, sua ordem original no HTML é mantida.

.flex-item {
    order: 0; /* Valor padrão. Pode ser qualquer número inteiro (positivo ou negativo) */
}

Exemplo: `order`

A ordem visual dos itens pode ser diferente da ordem no código HTML.

Item 1 (Ordem 2)
Item 2 (Ordem 1)
Item 3 (Ordem 3)
Item 4 (Ordem -1)

O `order` é útil para reordenar elementos para diferentes tamanhos de tela (responsividade) sem ter que mudar a estrutura HTML.

8.2. `flex-grow`

A propriedade **`flex-grow`** define a capacidade de um item flex de **crescer** e ocupar qualquer espaço extra disponível no contêiner flex, ao longo do eixo principal.

  • Valor padrão: `0` (o item não cresce, mesmo que haja espaço sobrando).
  • Valores maiores que `0`: O item pode crescer. O valor numérico representa uma proporção. Se dois itens tiverem `flex-grow: 1;`, eles crescerão igualmente. Se um tiver `flex-grow: 2;` e outro `flex-grow: 1;`, o primeiro ocupará o dobro do espaço extra que o segundo.
.flex-item {
    flex-grow: 0; /* Valor padrão. Pode ser qualquer número não negativo */
}

Exemplo: `flex-grow`

Itens crescem para preencher o espaço restante, de acordo com sua proporção `flex-grow`.

Item 1 (grow: 1)
Item 2 (grow: 0)
Item 3 (grow: 2)

Usar `flex-grow` é excelente para criar layouts fluidos onde os elementos se expandem para ocupar o espaço disponível, garantindo que o layout preencha a tela.

8.3. `flex-shrink`

A propriedade **`flex-shrink`** define a capacidade de um item flex de **encolher** se não houver espaço suficiente no contêiner para todos os itens em seu tamanho base. Isso evita que os itens transbordem do contêiner.

  • Valor padrão: `1` (o item pode encolher).
  • Valor `0`: O item não encolhe, mantendo seu tamanho original (ou `flex-basis`). Isso pode causar um transbordamento se o contêiner for muito pequeno.
  • Valores maiores que `1`: O item encolherá mais rapidamente que outros itens com `flex-shrink: 1;`.
.flex-item {
    flex-shrink: 1; /* Valor padrão. Pode ser qualquer número não negativo */
}

Exemplo: `flex-shrink`

8.4. `flex-basis`

A propriedade **`flex-basis`** define o **tamanho inicial padrão** de um item flex antes que qualquer espaço restante seja distribuído (ou removido) pelo `flex-grow` ou `flex-shrink`.

Ela pode ser definida com:

  • Um valor de comprimento (ex: `100px`, `50%`, `5rem`).
  • A palavra-chave `auto` (padrão): Usa o tamanho do conteúdo do item, ou o valor da propriedade `width` (ou `height` se `flex-direction` for `column`).
  • A palavra-chave `content`: Baseia o tamanho no tamanho intrínseco do conteúdo (similar ao `auto`, mas mais específico).
.flex-item {
    flex-basis: auto; /* Valor padrão. Pode ser uma unidade de comprimento ou 'content' */
}

Exemplo: `flex-basis`

Itens começam com o tamanho definido por `flex-basis`, antes de crescerem ou encolherem.

Auto

`flex-basis` é essencial para definir o "ponto de partida" dos itens em termos de tamanho, permitindo que Flexbox distribua o espaço a partir daí.

8.5. `flex` (shorthand)

A propriedade **`flex`** é uma propriedade de atalho (shorthand) para `flex-grow`, `flex-shrink` e `flex-basis`, nesta ordem.

Ela é a maneira mais comum e recomendada de definir as propriedades de flexibilidade de um item, pois controla como ele cresce, encolhe e qual é seu tamanho inicial.

A sintaxe completa é: `flex: ;`

  • `flex: 1;` é um valor comum e significa `flex: 1 1 0%;` (o item cresce, encolhe e tem base 0).
  • `flex: auto;` significa `flex: 1 1 auto;` (o item cresce, encolhe e tem base `auto`).
  • `flex: none;` significa `flex: 0 0 auto;` (o item não cresce, não encolhe e tem base `auto`).
  • `flex: 0 0 200px;` significa que o item não cresce, não encolhe, e terá uma largura fixa de 200px.
.flex-item {
    flex: 1; /* Um valor comum, faz o item ser flexível */
    /* Equivalente a: flex-grow: 1; flex-shrink: 1; flex-basis: 0%; */
}

Exemplo: `flex` Shorthand

Dominar a propriedade `flex` é um dos passos mais importantes para controlar o comportamento de dimensionamento dos seus itens flexíveis.

8.6. `align-self`

A propriedade **`align-self`** permite que um item flex individual sobrescreva o valor de `align-items` definido no contêiner flex para aquele item específico. Ela alinha um único item ao longo do eixo transversal.

Aceita os mesmos valores de `align-items`:

  • auto (padrão): O item herda o valor de `align-items` do seu pai ou `stretch` se não houver um.
  • flex-start: Alinha o item no início do eixo transversal.
  • flex-end: Alinha o item no final do eixo transversal.
  • center: Alinha o item no centro do eixo transversal.
  • baseline: Alinha o item pela linha de base do seu conteúdo.
  • stretch: O item estica para preencher o contêiner ao longo do eixo transversal.
.flex-item.especial {
    align-self: center; /* Sobrescreve o align-items do contêiner para este item */
}

9. Exemplos Práticos

Agora que você conhece as propriedades individuais do Flexbox, vamos ver como elas podem ser combinadas para resolver problemas de layout comuns de forma elegante.

9.1. Centralizando Elementos

Uma das tarefas mais comuns e simplificadas com Flexbox é a centralização de elementos, tanto horizontalmente quanto verticalmente. Isso é feito combinando `justify-content` e `align-items`.

.centralizar-tudo {
    display: flex;
    justify-content: center; /* Centraliza horizontalmente no eixo principal */
    align-items: center; /* Centraliza verticalmente no eixo transversal */
    height: 200px; /* Precisa de uma altura para visualizar o alinhamento vertical */
    border: 2px dashed #3498db;
}

9.2. Cabeçalho Responsivo

Criar um cabeçalho com logo, navegação e talvez um botão, e fazer com que eles se ajustem em diferentes tamanhos de tela.

.header {
    display: flex;
    justify-content: space-between; /* Logo em um lado, navegação no outro */
    align-items: center; /* Centraliza itens verticalmente no cabeçalho */
    background-color: #2c3e50;
    padding: 10px 20px;
    color: #fff;
    flex-wrap: wrap; /* Permite que os itens quebrem a linha em telas menores */
}

.header .logo {
    font-size: 1.5em;
    font-weight: bold;
    flex-shrink: 0; /* Impede que a logo encolha demais */
}

.header nav ul {
    list-style: none;
    padding: 0;
    margin: 0;
    display: flex; /* Torna a lista de navegação um contêiner flex */
    gap: 15px; /* Espaçamento entre os links */
    flex-wrap: wrap; /* Links da navegação podem quebrar em linhas */
    justify-content: center; /* Centraliza links quando quebram */
    flex-grow: 1; /* Permite que a navegação ocupe espaço extra */
}

.header nav ul li a {
    color: #fff;
    text-decoration: none;
    padding: 5px 10px;
    border-radius: 3px;
    transition: background-color 0.3s ease;
}
.header nav ul li a:hover {
    background-color: #34495e;
}

9.3. Grid de Cards Responsivo

Usando Flexbox para criar um layout de cards que se adapta a diferentes números de colunas automaticamente.

.card-grid {
    display: flex;
    flex-wrap: wrap; /* Essencial para que os cards quebrem em novas linhas */
    gap: 20px; /* Espaçamento entre os cards */
    justify-content: center; /* Centraliza os cards quando há poucos na última linha */
}

.card {
    flex: 1 1 calc(33% - 20px); /* flex-grow, flex-shrink, flex-basis */
    /* flex-basis: calcula 33% da largura do pai menos o gap para 3 colunas */
    min-width: 280px; /* Impede que os cards fiquem muito pequenos */
    box-shadow: 0 2px 5px rgba(0,0,0,0.1);
    border-radius: 8px;
    padding: 20px;
    background-color: #fff;
    text-align: center;
}

/* Media query para telas menores */
@media (max-width: 768px) {
    .card {
        flex-basis: calc(50% - 20px); /* Duas colunas em telas médias */
    }
}

@media (max-width: 480px) {
    .card {
        flex-basis: 100%; /* Uma coluna em telas muito pequenas */
    }
}
