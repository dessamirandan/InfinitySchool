Apostila CSS Grid: Guia Completo para Iniciantes

Aprenda a criar layouts poderosos e responsivos com CSS Grid.

Introdução ao CSS Grid

Bem-vindo(a) ao mundo do CSS Grid! Esta apostila foi elaborada para te guiar do básico ao avançado, com exemplos claros e visuais que te ajudarão a dominar esta poderosa ferramenta de layout do CSS.

O que é CSS Grid?

O **CSS Grid Layout**, ou simplesmente Grid, é um sistema de layout **bidimensional** para a web. Isso significa que ele pode organizar seus elementos tanto em **linhas** quanto em **colunas** ao mesmo tempo. Pense em uma folha de papel quadriculada, onde você pode desenhar e posicionar seus elementos em células específicas.

Ao contrário do Flexbox (que é unidimensional e serve para organizar itens em uma única linha ou coluna), o Grid é perfeito para construir a **estrutura principal** de páginas web, como o layout de um site inteiro (cabeçalho, navegação, conteúdo, rodapé).

Por que usar CSS Grid?

  • **Layouts Complexos:** Crie estruturas de página complicadas de forma mais simples e intuitiva.
  • **Responsividade Fácil:** O Grid se adapta muito bem a diferentes tamanhos de tela (celulares, tablets, desktops), tornando a criação de sites que funcionam em qualquer dispositivo muito mais eficiente.
  • **Controle Total:** Tenha controle preciso sobre onde cada elemento será posicionado e qual o tamanho que ele ocupará.
  • **Código Limpo:** Ele separa a forma como seu conteúdo está escrito (HTML) da forma como ele é apresentado (CSS), deixando seu código mais organizado e fácil de entender.
  • **Manutenção Simples:** Layouts feitos com Grid são geralmente mais fáceis de modificar e atualizar no futuro.

Grid vs. Flexbox: Qual a diferença?

Essa é uma dúvida comum para iniciantes. Veja um resumo das principais diferenças:

Característica CSS Grid CSS Flexbox
**Dimensão** **Bidimensional** (linhas e colunas juntas) **Unidimensional** (apenas linha OU apenas coluna)
**Uso Ideal** Para o **layout geral da página** (ex: cabeçalho, sidebar, conteúdo principal, rodapé) Para **alinhar e distribuir itens dentro de um componente** (ex: botões em uma barra de navegação, itens em um cartão de produto)
**Controle** Principalmente no **elemento pai (container)**, definindo a grade para todos os filhos. Principalmente nos **elementos filhos**, para o alinhamento individual.

Em resumo: use **Grid para o esqueleto** da sua página e **Flexbox para arrumar os elementos dentro dos "ossos"** desse esqueleto. Eles são melhores amigos e se complementam!

Começando com CSS Grid

Para começar a usar o CSS Grid, você precisa de dois tipos de elementos:

  • Um **container Grid**: O elemento pai que vai "segurar" todos os outros elementos e onde você vai definir a grade.
  • Os **itens Grid**: Os elementos filhos que serão organizados dentro dessa grade.

O Container Grid: A base de tudo!

Para transformar qualquer elemento HTML em um container Grid, você usa a propriedade CSS **`display: grid;`**. É a primeira coisa que você precisa fazer!

Exemplo Básico:

HTML:

<div class="grid-container">
  <div class="grid-item">1</div>
  <div class="grid-item">2</div>
  <div class="grid-item">3</div>
  <div class="grid-item">4</div>
  <div class="grid-item">5</div>
  <div class="grid-item">6</div>
</div>

CSS:

.grid-container {
  display: grid; /* Esta linha transforma o elemento em um container Grid! */
  /* Outras propriedades Grid virão aqui para definir linhas e colunas */
}

.grid-item {
  background-color: rgba(100, 149, 237, 0.8); /* Azul-milho para os itens */
  border: 1px solid rgba(0, 0, 0, 0.8);
  padding: 20px;
  font-size: 30px;
  text-align: center;
  color: white;
}

Propriedades do Container Grid

As propriedades que você aplica ao **container Grid** (o elemento pai com `display: grid;`) são as mais importantes para definir a estrutura geral do seu layout.

1. `grid-template-columns`: Definindo as Colunas!

Esta propriedade permite que você defina **quantas colunas** você quer ter no seu Grid e **qual a largura** de cada uma delas.

Exemplos:

Três colunas com largura fixa (em pixels):

.grid-container {
  display: grid;
  grid-template-columns: 100px 100px 100px; /* Três colunas, cada uma com 100 pixels de largura */
}

Aqui, cada item ocupa sua célula de 100px. Se houver mais itens que células, o Grid criará novas linhas.

Três colunas de largura igual, usando a unidade `fr` (fraction):

.grid-container {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr; /* Três colunas, cada uma ocupando 1/3 do espaço disponível */
}

A unidade **`fr`** (fraction unit) é fantástica para layouts flexíveis. Ela distribui o espaço **disponível** no container Grid em "frações". Se você tem `1fr 1fr 1fr`, cada coluna recebe uma fração (1/3) do espaço que sobra. É ideal para designs responsivos.

Misturando unidades e a função `repeat()`:

.grid-container {
  display: grid;
  /* Uma coluna de 100px, as outras duas ocupam o resto do espaço igualmente */
  grid-template-columns: 100px 1fr 1fr;
}

/* Usando a função repeat() para simplificar: */
.grid-container-repeat {
  grid-template-columns: repeat(3, 1fr); /* O mesmo que 1fr 1fr 1fr */
}

A função **`repeat()`** é um atalho útil. O primeiro argumento é o número de repetições, e o segundo é o valor da trilha (coluna ou linha) a ser repetida.

A função `minmax()` para responsividade avançada:

.grid-container {
  display: grid;
  /* Colunas com no mínimo 100px, mas que se expandem para ocupar o espaço disponível */
  grid-template-columns: repeat(auto-fill, minmax(100px, 1fr));
}

A função **`minmax(min, max)`** define um tamanho mínimo e máximo para uma trilha. Isso é super poderoso para layouts responsivos!

  • **`auto-fill`**: Tenta preencher a linha com o máximo de colunas que couberem.
  • **`minmax(100px, 1fr)`**: Cada coluna terá no mínimo 100px. Se houver espaço extra, elas se expandem igualmente, ocupando o restante do espaço.

2. `grid-template-rows`: Definindo as Linhas!

Funciona exatamente como `grid-template-columns`, mas para as alturas das suas linhas. Você pode usar pixels, porcentagens, `em`, `rem` ou `fr`.

Exemplo:

CSS:

.grid-container {
  display: grid;
  grid-template-columns: auto auto auto; /* Três colunas automáticas */
  grid-template-rows: 80px 200px; /* Duas linhas: a primeira com 80px, a segunda com 200px */
}

3. `grid-template-areas`: Dando nomes às Áreas do Layout!

Esta é uma das propriedades mais visuais e intuitivas do Grid. Ela permite que você dê **nomes** para as áreas do seu layout e depois posicione os itens Grid usando esses nomes. É como desenhar seu layout com texto!

Exemplo de um Layout de Página:

HTML: (Os itens que serão posicionados)

<div class="grid-container-areas">
  <header class="item-header">Header</header>
  <nav class="item-menu">Menu</nav>
  <main class="item-main">Conteúdo Principal</main>
  <aside class="item-right">Lateral Direita</aside>
  <footer class="item-footer">Rodapé</footer>
</div>

CSS:

.grid-container-areas {
  display: grid;
  /* Definindo as áreas do layout */
  grid-template-areas:
    'header header header' /* A primeira linha: 'header' ocupa 3 colunas */
    'menu   main   right'  /* A segunda linha: 'menu', 'main', 'right' */
    'footer footer footer';/* A terceira linha: 'footer' ocupa 3 colunas */
  
  /* Definindo as larguras das colunas para essas áreas */
  grid-template-columns: 150px 1fr 100px; /* 150px para 'menu', 1fr para 'main', 100px para 'right' */
  
  /* Definindo as alturas das linhas para essas áreas */
  grid-template-rows: 80px 1fr 60px; /* 80px para 'header', 1fr para 'main' (e menu/right), 60px para 'footer' */
  
  height: 500px; /* Apenas para visualizar o Grid */
  border: 1px solid #ccc;
  grid-gap: 10px; /* Espaçamento entre as áreas */
}

/* Associando os itens HTML às áreas nomeadas */
.item-header  { grid-area: header; background-color: #ffd700; } /* Dourado */
.item-menu    { grid-area: menu;   background-color: #ff6347; } /* Tomate */
.item-main    { grid-area: main;   background-color: #66cdaa; } /* Verde-água */
.item-right   { grid-area: right;  background-color: #ffa07a; } /* Salmão claro */
.item-footer  { grid-area: footer; background-color: #add8e6; } /* Azul claro */

/* Estilos básicos para os itens */
.item-header, .item-menu, .item-main, .item-right, .item-footer {
  padding: 20px;
  text-align: center;
  font-weight: bold;
  color: #333;
}

**Explicação Detalhada:**

  • Cada string em `grid-template-areas` representa uma **linha** do seu Grid.
  • As palavras dentro de cada string representam as **áreas** nessa linha.
  • Repetir o nome da área (`header header header`) faz com que essa área **se estenda** por várias colunas.
  • Um `.` (ponto) pode ser usado para deixar uma célula **vazia**. Ex: `'header . header'`
  • Depois de definir as áreas, você usa `grid-template-columns` e `grid-template-rows` para dizer as **dimensões** das suas colunas e linhas.
  • Finalmente, você associa cada elemento HTML a uma área usando a propriedade **`grid-area`** no CSS.

Este método é incrivelmente poderoso para criar layouts complexos e facilmente compreensíveis. Se você precisar mudar o layout, basta alterar a string em `grid-template-areas`!

4. `grid-gap` (ou `column-gap` e `row-gap`): Criando Espaços!

Estas propriedades definem o espaço (calha) entre as células do seu Grid. É como a "margem" interna entre seus itens.

  • `grid-gap`: Um atalho que define o espaço tanto entre linhas quanto entre colunas.
  • `row-gap`: Apenas o espaço entre as linhas.
  • `column-gap`: Apenas o espaço entre as colunas.

Exemplo:

CSS:

.grid-container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-gap: 20px; /* Adiciona 20px de espaço entre todas as células (linhas e colunas) */
}

.grid-container-specific-gap {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  row-gap: 10px;    /* 10px de espaço entre as linhas */
  column-gap: 30px; /* 30px de espaço entre as colunas */
}

5. `justify-items` e `align-items`: Alinhando Itens Dentro das Células

Imagine que cada célula do Grid é um pequeno espaço. Estas propriedades controlam como os itens se alinham **dentro de suas próprias células**, ao longo do eixo horizontal (`justify-items`) e vertical (`align-items`).

  • **`justify-items` (alinhamento horizontal dentro da célula):**
    • `start`: Alinha o item ao início da célula (esquerda).
    • `end`: Alinha o item ao final da célula (direita).
    • `center`: Centraliza o item na célula (horizontalmente).
    • `stretch` (padrão): Estica o item para preencher toda a largura da célula.
  • **`align-items` (alinhamento vertical dentro da célula):**
    • `start`: Alinha o item ao topo da célula.
    • `end`: Alinha o item ao fundo da célula.
    • `center`: Centraliza o item na célula (verticalmente).
    • `stretch` (padrão): Estica o item para preencher toda a altura da célula.

Exemplo:

CSS:

.grid-container {
  display: grid;
  grid-template-columns: 150px 150px 150px; /* Colunas fixas para demonstrar o alinhamento */
  grid-template-rows: 100px 100px; /* Linhas fixas */
  justify-items: center; /* Centraliza todos os itens horizontalmente dentro de suas células */
  align-items: center;   /* Centraliza todos os itens verticalmente dentro de suas células */
  border: 1px solid #ccc;
}

6. `justify-content` e `align-content`: Alinhando o Grid Completo!

Estas propriedades são usadas quando o seu **Grid como um todo** (o container) não ocupa 100% da largura ou altura disponível do seu elemento pai. Elas controlam como o Grid inteiro é alinhado dentro desse espaço extra.

  • **`justify-content` (alinhamento horizontal do Grid):**
    • `start`: Alinha o Grid ao início do container pai (esquerda).
    • `end`: Alinha o Grid ao final do container pai (direita).
    • `center`: Centraliza o Grid no container pai (horizontalmente).
    • `space-around`: Distribui o espaço extra ao redor dos trilhos.
    • `space-between`: Distribui o espaço extra entre os trilhos, com os primeiros e últimos trilhos nas bordas.
    • `space-evenly`: Distribui o espaço extra de forma igual entre os trilhos e nas bordas.
  • **`align-content` (alinhamento vertical do Grid):**
    • `start`: Alinha o Grid ao topo do container pai.
    • `end`: Alinha o Grid ao fundo do container pai.
    • `center`: Centraliza o Grid verticalmente no container pai.
    • `space-around`, `space-between`, `space-evenly`: Funcionam da mesma forma que em `justify-content`, mas no eixo vertical.

Exemplo:

CSS:

.grid-container-content {
  display: grid;
  grid-template-columns: 100px 100px; /* Duas colunas de 100px */
  grid-template-rows: 80px 80px;      /* Duas linhas de 80px */
  
  width: 400px;  /* O container Grid é menor que a área disponível do pai */
  height: 300px; /* O container Grid é menor que a área disponível do pai */
  
  justify-content: center; /* Centraliza o Grid inteiro horizontalmente */
  align-content: center;   /* Centraliza o Grid inteiro verticalmente */
  
  border: 2px dashed red; /* Borda para visualizar o container */
}

7. `grid-auto-columns` e `grid-auto-rows`: Células "Extras"

Às vezes, você pode ter mais itens Grid do que o número de colunas ou linhas que você definiu explicitamente com `grid-template-columns` ou `grid-template-rows`. Quando isso acontece, o Grid cria "células extras" automaticamente.

Essas propriedades permitem que você defina o tamanho dessas trilhas (colunas ou linhas) que são criadas "na hora".

  • `grid-auto-columns`: Define a largura das colunas criadas automaticamente.
  • `grid-auto-rows`: Define a altura das linhas criadas automaticamente.

Exemplo:

HTML:

<div class="grid-container-auto">
  <div class="grid-item">1</div>
  <div class="grid-item">2</div>
  <div class="grid-item">3</div>
  <div class="grid-item">4</div>
  <div class="grid-item">5</div>
  <div class="grid-item">6</div>
  <div class="grid-item">7</div> <!-- Este item vai para uma linha criada automaticamente -->
</div>

CSS:

.grid-container-auto {
  display: grid;
  grid-template-columns: repeat(3, 1fr); /* Explicitamente 3 colunas */
  grid-auto-rows: 50px; /* Linhas criadas automaticamente terão 50px de altura */
  border: 1px solid #ccc;
}

8. `grid-auto-flow`: Como os Itens se Encaixam Automaticamente

Quando você não posiciona explicitamente cada item Grid, o `grid-auto-flow` decide como eles devem ser preenchidos no Grid. É o "algoritmo" de preenchimento padrão.

  • `row` (padrão): Os itens são preenchidos linha por linha, da esquerda para a direita, adicionando novas linhas conforme necessário.
  • `column`: Os itens são preenchidos coluna por coluna, de cima para baixo, adicionando novas colunas conforme necessário.
  • `dense`: Tenta preencher espaços vazios no Grid com itens que "cabem", mesmo que isso mude a ordem visual dos itens em relação à ordem no HTML. Ajuda a otimizar o espaço.

Exemplo:

CSS:

.grid-container-flow {
  display: grid;
  grid-template-columns: repeat(3, 1fr); /* 3 colunas */
  grid-auto-flow: column; /* Itens preenchem coluna por coluna */
  height: 200px; /* Para visualizar o fluxo em colunas */
  border: 1px solid #ccc;
}

Propriedades dos Itens Grid

As propriedades dos **itens Grid** são aplicadas diretamente aos elementos filhos do seu container Grid. Elas permitem que você controle a posição e o tamanho de cada item individualmente dentro da grade que você definiu.

1. `grid-column-start`, `grid-column-end`, `grid-row-start`, `grid-row-end`: Posicionamento Preciso!

Estas propriedades são como coordenadas que dizem a um item exatamente onde ele deve começar e terminar no Grid. Para usá-las, você precisa entender o conceito de **linhas do Grid**.

Entendendo as Linhas do Grid:

O Grid cria linhas numeradas entre suas colunas e linhas. Se você tem 3 colunas, terá 4 linhas verticais (chamadas de linha 1, linha 2, linha 3 e linha 4). Se tiver 2 linhas horizontais, terá 3 linhas horizontais (linha 1, linha 2 e linha 3).

Veja a imagem para entender as linhas:

As linhas numeradas são a sua referência para posicionar os itens.

  • `grid-column-start`: A linha **vertical** onde o item começa.
  • `grid-column-end`: A linha **vertical** onde o item termina.
  • `grid-row-start`: A linha **horizontal** onde o item começa.
  • `grid-row-end`: A linha **horizontal** onde o item termina.

Exemplo:

HTML:

<div class="grid-container-item-pos">
  <div class="grid-item item-a">Item A</div>
  <div class="grid-item item-b">Item B</div>
  <div class="grid-item item-c">Item C</div>
  <div class="grid-item item-d">Item D</div>
  <div class="grid-item item-e">Item E</div>
  <div class="grid-item item-f">Item F</div>
</div>

CSS:

.grid-container-item-pos {
  display: grid;
  grid-template-columns: repeat(3, 1fr); /* 3 colunas de largura igual */
  grid-template-rows: repeat(2, 100px); /* 2 linhas de 100px */
  border: 1px solid #ccc;
}

.item-a {
  background-color: #ffb6c1; /* Rosa claro */
  grid-column-start: 1; /* Começa na linha vertical 1 */
  grid-column-end: 3;   /* Termina na linha vertical 3 (ocupando 2 colunas: da 1 para a 2) */
  grid-row-start: 1;    /* Começa na linha horizontal 1 */
  grid-row-end: 2;      /* Termina na linha horizontal 2 (ocupando 1 linha) */
}

.item-b {
  background-color: #90ee90; /* Verde claro */
  grid-column-start: 3; /* Começa na linha vertical 3 */
  grid-column-end: 4;   /* Termina na linha vertical 4 (ocupando 1 coluna) */
  grid-row-start: 1;    /* Começa na linha horizontal 1 */
  grid-row-end: 3;      /* Termina na linha horizontal 3 (ocupando 2 linhas) */
}

Resultado Visual (grid-item-pos-example):

Repare como o Item A se estende por duas colunas e o Item B por duas linhas. O Grid automaticamente ajusta os outros itens para preencher os espaços vazios, se houver.

2. `grid-column` e `grid-row`: Atalhos Mais Curtos!

São atalhos para as propriedades `*-start` e `*-end`. São mais concisos e amplamente usados.

  • `grid-column: start / end;`
  • `grid-row: start / end;`

Você também pode usar a palavra-chave **`span`** para indicar quantas linhas/colunas o item deve "abrangir" a partir do seu ponto de início.

Exemplo:

CSS:

.item-a-shortcut {
  grid-column: 1 / span 2; /* Começa na linha vertical 1 e abrange 2 colunas (vai até a linha 3) */
  grid-row: 1 / 2;         /* Começa na linha horizontal 1 e termina na linha 2 (abrangendo 1 linha) */
}

.item-b-shortcut {
  grid-column: 3 / 4;      /* Começa na linha 3 e termina na linha 4 (ocupa a última coluna) */
  grid-row: 1 / span 2;    /* Começa na linha 1 e abrange 2 linhas (vai até a linha 3) */
}

Resultado Visual: (Mesmo resultado do exemplo anterior, mas com código mais limpo!)

Item A (atalho)
Item B (atalho)
Item C
Item D
Item E
Item F

Usar `span` é super útil quando você sabe a partir de onde um item deve começar e por quantas células ele precisa se estender.

3. `grid-area`: Posicionando pelo Nome ou Pelas Linhas (Atalho Super!)

Esta propriedade tem duas finalidades:

  1. **Posicionar um item usando os nomes de área** que você definiu com `grid-template-areas` (essa é a forma mais comum e recomendada).
  2. Como um atalho para `grid-row-start / grid-column-start / grid-row-end / grid-column-end` (útil se você não estiver usando `grid-template-areas`).

Exemplo 1: Usando Nomes de Área (Recomendado!)

(Este é o mesmo exemplo que vimos em `grid-template-areas`, mas agora focando na propriedade `grid-area` dos itens.)

CSS: (Fragmento, o container completo está na seção 3 do container)

/* No container: */
.grid-container-areas {
  /* ... */
  grid-template-areas:
    'header header header'
    'menu   main   right'
    'footer footer footer';
  /* ... */
}

/* Nos itens: */
.item-header  { grid-area: header; } /* Este item vai para a área nomeada 'header' */
.item-menu    { grid-area: menu;   }
.item-main    { grid-area: main;   }
.item-right   { grid-area: right;  }
.item-footer  { grid-area: footer; }

Resultado Visual: (Mesmo do exemplo de `grid-template-areas`)

Header (com grid-area)
Menu (com grid-area)
Conteúdo Principal (com grid-area)
Lateral Direita (com grid-area)
Rodapé (com grid-area)

Essa é a forma mais legível e poderosa de posicionar itens em layouts complexos. Basta apontar o item para o nome da área que você "desenhou".

Exemplo 2: Usando `grid-area` como Atalho para Linhas

CSS:

.grid-container-area-shortcut {

4. `justify-self` e `align-self`: Alinhamento Individual!

Estas propriedades são aplicadas a **itens individuais** e servem para **sobrescrever** o alinhamento que foi definido no container Grid (com `justify-items` e `align-items`).

  • `justify-self`: Alinhamento horizontal do **item específico** dentro de sua célula.
  • `align-self`: Alinhamento vertical do **item específico** dentro de sua célula.

Os valores são os mesmos: `start`, `end`, `center`, `stretch`.

Exemplo:

CSS:

.grid-container-self {
  display: grid;
  grid-template-columns: repeat(3, 150px);
  grid-template-rows: repeat(2, 100px);
  justify-items: stretch; /* Padrão: itens esticam horizontalmente */
  align-items: stretch;   /* Padrão: itens esticam verticalmente */
  border: 1px solid #ccc;
}

.item-specific-align {
  background-color: #b0e0e6; /* Azul claro */
  justify-self: end;   /* Este item específico vai para o final da sua célula (direita) */
  align-self: center;  /* Este item específico vai para o centro da sua célula (verticalmente) */
}

Mesmo que o container esteja esticando todos os itens, o Item 2 ignora essa regra e se alinha conforme suas próprias propriedades `justify-self` e `align-self`.

1. Layout de 2 Colunas (Sidebar e Conteúdo Principal)

Um layout clássico, perfeito para blogs ou painéis administrativos.

HTML:

<div class="layout-2-cols">
  <aside class="sidebar">
    <h3>Navegação</h3>
    <ul>
      <li>Item Menu 1</li>
      <li>Item Menu 2</li>
      <li>Item Menu 3</li>
    </ul>
  </aside>
  <section class="main-content-2cols">
    <h3>Conteúdo Principal</h3>
    <p>Este é o conteúdo principal da página. Ele ocupa a maior parte do espaço e se expande conforme necessário.</p>
    <p>É um exemplo de layout de duas colunas, onde uma é fixa e a outra é flexível.</p>
  </section>
</div>

CSS:

.layout-2-cols {
  display: grid;
  grid-template-columns: 200px 1fr; /* Sidebar de 200px, conteúdo principal ocupa o resto (1fr) */
  grid-gap: 30px; /* Espaço entre o sidebar e o conteúdo */
  height: 250px; /* Apenas para visualização */
  border: 1px solid #ccc;
  align-items: start; /* Alinha os itens ao topo */
}

.sidebar {
  background-color: #f0f8ff; /* Alice Blue */
  padding: 15px;
  border-right: 1px solid #ccc;
}

.main-content-2cols {
  background-color: #f5fffa; /* Mint Cream */
  padding: 15px;

2. Layout Responsivo com `auto-fit` e `minmax` (Galeria de Cards)

Este é o segredo para criar galerias de imagens ou listas de produtos que se adaptam automaticamente a qualquer tamanho de tela. Menos media queries para você!

HTML:

<div class="grid-responsive-cards">
  <div class="card">Card 1</div>
  <div class="card">Card 2</div>
  <div class="card">Card 3</div>
  <div class="card">Card 4</div>
  <div class="card">Card 5</div>
  <div class="card">Card 6</div>
  <div class="card">Card 7</div>
  <div class="card">Card 8</div>
</div>

CSS:

.grid-responsive-cards {
  display: grid;
  /* Cria o máximo de colunas que puderem caber.
     Cada coluna terá NO MÍNIMO 200px.
     Se houver espaço extra, ele será distribuído igualmente (1fr). */
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  grid-gap: 20px; /* Espaço entre os cards */
  padding: 20px;
  background-color: #fafad2; /* Light Goldenrod Yellow */
  border: 1px solid #ccc;
}

.card {
  background-color: #add8e6; /* Light Blue */
  padding: 30px;
  text-align: center;
  border: 1px solid #333;
  box-shadow: 2px 2px 5px rgba(0,0,0,0.2);
}

3. Layout Completo de Página com Áreas Nomeadas

Este é o layout mais comum para sites: um cabeçalho, navegação, conteúdo principal, talvez uma barra lateral e um rodapé.

HTML:

<div class="page-full-layout">
  <header class="page-header">Cabeçalho do Site</header>
  <nav class="page-nav">Navegação Principal</nav>
  <main class="page-main">
    <h2>Seja Bem-vindo!</h2>
    <p>Este é o conteúdo principal da sua página. Com CSS Grid e áreas nomeadas, organizar o layout é muito fácil e intuitivo.</p>
    <p>Você pode mover as seções do seu site apenas mudando as strings em `grid-template-areas`!</p>
  </main>
  <aside class="page-aside">Barra Lateral</aside>
  <footer class="page-footer">Rodapé do Site © 2025</footer>
</div>

CSS:

.page-full-layout {
  display: grid;
  grid-template-areas:
    "header header header" /* Header ocupa toda a largura da primeira linha */
    "nav    main   aside"  /* Nav à esquerda, Main no centro, Aside à direita */
    "footer footer footer";/* Footer ocupa toda a largura da última linha */
  
  /* Definindo as larguras das colunas: */
  grid-template-columns: 150px 1fr 200px; /* Nav: 150px, Main: flexível (resto), Aside: 200px */
  
  /* Definindo as alturas das linhas: */
  grid-template-rows: 80px 1fr 60px; /* Header: 80px, Main (e nav/aside): flexível, Footer: 60px */
  
  height: 100vh; /* Faz o layout ocupar 100% da altura da tela */
  grid-gap: 15px; /* Espaço entre as seções */
  font-family: sans-serif;
  color: #333;
}

/* Atribuindo as áreas aos elementos */
.page-header  { grid-area: header; background-color: #6a5acd; color: white; text-align: center; padding: 20px;} /* Slate Blue */
.page-nav     { grid-area: nav;    background-color: #e6e6fa; padding: 20px;} /* Lavender */
.page-main    { grid-area: main;   background-color: #f0f8ff; padding: 20px;} /* Alice Blue */
.page-aside   { grid-area: aside;  background-color: #fff0f5; padding: 20px;} /* Lavender Blush */
.page-footer  { grid-area: footer; background-color: #8a2be2; color: white; text-align: center; padding: 10px;} /* Blue Violet */

/* Estilos adicionais para centralizar conteúdo nos itens */
.page-header h1, .page-footer p {
    margin: 0;
}

Ferramentas de Desenvolvimento

Dominar o CSS Grid fica muito mais fácil com as ferramentas certas! Todos os navegadores modernos (Chrome, Firefox, Edge, Safari) oferecem recursos incríveis para inspecionar e depurar seus layouts Grid.

Como Acessar:

1. Abra o seu site no navegador.

2. Clique com o botão direito do mouse em qualquer lugar da página e selecione **"Inspecionar"** ou **"Inspecionar Elemento"** (o nome pode variar um pouco).

3. Na aba **"Elements"** (ou "Elementos"), selecione o seu container Grid (o elemento que tem `display: grid;`).

4. Procure por um pequeno ícone de "Grid" ou um seletor na aba **"Styles"** (ou "Estilos") ou em uma aba dedicada a "Layout" (no Firefox). Ao clicar nele, você verá as linhas do seu Grid e os nomes das áreas sobrepostas na sua página! Isso é extremamente útil para entender o que está acontecendo.

O inspetor de Grid do navegador mostra as linhas e áreas, ajudando na depuração.

**Dica:** O **Firefox Developer Tools** é frequentemente elogiado por ter uma das melhores visualizações e controles para CSS Grid. Vale a pena experimentar!

Conclusão

Parabéns! Você deu um grande passo para se tornar um desenvolvedor web completo ao mergulhar no CSS Grid. Lembre-se, aprender é um processo contínuo.

Recapitulando os pontos mais importantes:

  • O **Grid** é para layouts **bidimensionais** (linhas e colunas), ideal para a estrutura principal da página.
  • Comece sempre com `display: grid;` no seu **container**.
  • Use `grid-template-columns` e `grid-template-rows` para definir as dimensões do seu Grid.
  • A unidade `fr` e a função `minmax()` são seus melhores amigos para layouts **flexíveis e responsivos**.
  • `grid-template-areas` é a forma mais poderosa e legível de desenhar layouts complexos.
  • Use `grid-gap` para adicionar espaçamento entre seus itens.
  • Entenda a diferença entre alinhar **itens dentro das células** (`justify-items`, `align-items`) e alinhar o **Grid completo** (`justify-content`, `align-content`).
  • Utilize as **ferramentas de desenvolvimento** do seu navegador para visualizar e depurar seu Grid. Elas são um salva-vidas!

Continue praticando! A melhor forma de aprender é construindo. Crie seus próprios layouts, tente replicar designs que você vê na web usando Grid. Você vai se surpreender com o que pode criar!
